# Lessons Evolve（经验三级进化）

本文件是一份跨 Agent 的通用规则，适用于支持 `AGENTS.md` 的 AI Agent 工具（如 OpenAI Codex CLI、OpenCode、MiMo Code 等）。

> 核心理念：一个坑踩一次写成 **lesson**，踩两次归纳为 **pattern**，踩三次晋升为 **instinct**。每次晋升都需要人工确认，防止错误经验扩散。

## 触发时机（严格限定）

仅在以下场景主动调用本规则：

- 本轮任务出现非显然修正：不是打字错误、不是 TDD 红绿循环中的预期失败、不是显而易见的漏配。
- 用户明确表达重复问题：例如“又遇到了”“上次也这样”“怎么又踩这个坑”。
- 故障恢复、根因分析、周报/月报/结项复盘后，用户要求沉淀经验。
- 用户主动要求：“总结下这次踩的坑”“把经验沉淀成规则”“检查有没有 lesson”。

以下场景不触发：普通编译报错、TDD 预期失败、明显 typo、每次任务都有的常规操作步骤。

## Step 0：环境检查

执行前先确认当前 Agent 可用能力，并据此选择数据通道：

- 长期记忆能力可用：优先写入长期记忆。
- 历史检索能力可用：用于检索过往 session 和 memory 中的同类 lesson。
- 文件搜索/读取能力可用：可读写当前项目 `.lessons-evolve/lessons.yaml`。
- 以上都不可用：在回复正文输出完整记录，让用户自行保存。

数据通道优先级：长期记忆能力 → 当前项目 `.lessons-evolve/lessons.yaml` → 回复正文。

## 执行流程

1. **识别问题**：提取当前问题的 `symptom`、`root_cause`、`solution`，并判断是否属于非显然修正；不属于则输出“经验检查完毕，本次无新经验需要记录”。
2. **搜索历史**：按可用能力检索历史 lesson；有历史检索和文件访问时同时搜索，取并集。
3. **判断晋升等级**：
   - 无命中：记录为 **lesson**，写入所选数据通道，并通知用户存储位置与证据 ID。
   - 命中 1 次且拒绝计数 `< 2`：询问用户是否创建 **pattern**，可落到当前 Agent 支持的 skill、rule 或 `AGENTS.md` 片段。
   - 命中 2 次以上且拒绝计数 `< 2`：询问用户是否固化为 **instinct**，例如流程 checklist、默认检查项或长期知识。
   - 拒绝计数 `>= 2`：不再提醒同一 lesson 晋升，只说明之前已确认不处理。
4. **结果通知**：通知必输出，说明新增、跳过、拒绝、固化或无变化的结果，以及后续如何避免。

## 同类判定

满足以下任一条件，即判定为同类问题：

- 症状相同：报错信息、异常类型或可观察现象一致。
- 根因相同：底层原因一致。
- 解决方案相同：使用了同样的命令、配置或步骤修复。

## lesson 记录模板

```yaml
- lesson_id: "ev-YYYYMMDD-001"
  title: "一句话描述"
  symptom: "具体的现象/报错信息"
  root_cause: "根因"
  solution: "修复步骤"
  tags: ["keyword"]
  source_session: "来源会话标识"
  created_at: "YYYY-MM-DDTHH:mm:ssZ"
  promotion_count: 0
  last_hit_at: null
  status: "active"
```

## 防偏差与回退

- lesson → pattern、pattern → instinct 都必须先获得用户确认。
- 不强行合并“看起来相似但本质不同”的问题。
- 用户连续拒绝两次后，将该 lesson 标记为 `user_rejected`，不再重复提醒。
- 用户可说“撤销 {title} 的固化”“移除 {title} 的规则”“彻底忘掉 {title}”执行降级或删除。
- 每次晋升保留 lesson ID 和来源摘要，保证可追溯。

## 输出契约

- lesson：输出问题、根因、证据 ID、存储位置和后续处理。
- pattern：输出规则名称、来源证据和规则内容摘要。
- instinct：输出固化位置、生效范围和回退方式。
- 拒绝：输出拒绝计数与是否继续提醒。
- 无变化：输出“经验检查完毕，本次无新经验需要记录”。
